## Лабораторная работа №1

**Базовая формулировка:**
Реализовать межпроцессное взаимодействие через файл дескриптор (лучше через tcp сокет).
Нужно сделать 2 приложения (C/C++), одно с ролью клиента, второе с ролью сервера. Они должны обмениваться сообщениями ping (клиент), pong (ответ от сервера). Для безопасного взаимодействия, алгоритмика должна соответствовать диаграмме состояний (на примере файла, см. рисунок).

Для максимального балла только C/C++, только std и уделить внимание обработке ошибочных ситуаций.

---

Выше будет оцениваться, если будет написан простенький чат с возможностью получения истории переписки, где файловый дескриптор можно использовать как транспортную шину (в случае использования файла).

Ещё выше, если приложения будут собраны любой системой сборки (к примеру cmake или meson).
